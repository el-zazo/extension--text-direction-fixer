# Storage Schema

All data is persisted in `chrome.storage.local`. This document describes every storage key, its data type, and the shape of the stored objects.

---

## Storage Keys

| Key | Type | Description |
|---|---|---|
| `rtl_data` | `Page[]` | Array of page objects, each containing saved RTL selectors. |
| `rtl_theme` | `"dark" \| "light"` | Theme preference for the Options dashboard. Defaults to `"dark"`. |

---

## Data Types

### `Page`

Represents a single website (identified by URL) that has RTL selectors saved.

```typescript
interface Page {
  url: string;          // origin + pathname (e.g. "https://example.com/docs")
  pageEnabled: boolean; // Master switch: if false, no selectors are applied on this page
  createdAt: string;    // ISO 8601 timestamp of when this page was first saved
  selectors: Selector[];// Array of CSS selectors for this page
}
```

| Field | Type | Required | Description |
|---|---|---|---|
| `url` | `string` | Yes | The page URL, built as `window.location.origin + window.location.pathname`. Query strings and hashes are **not** included. |
| `pageEnabled` | `boolean` | Yes | When `true`, enabled selectors are automatically applied on page load. When `false`, no selectors are applied and a warning notification is shown. |
| `createdAt` | `string` | Yes | ISO 8601 date string (e.g. `"2025-03-15T14:30:00.000Z"`). Generated by `new Date().toISOString()`. |
| `selectors` | `Selector[]` | Yes | List of CSS selector entries. Can be empty (but pages with 0 selectors are filtered out of the Options list). |

### `Selector`

Represents a single CSS selector path that applies RTL styling to matching elements.

```typescript
interface Selector {
  id: string;        // crypto.randomUUID()
  path: string;      // CSS selector path (e.g. "div.content > p.text")
  enabled: boolean;  // Whether this selector is active
  createdAt: string; // ISO 8601 timestamp of when this selector was created
}
```

| Field | Type | Required | Description |
|---|---|---|---|
| `id` | `string` | Yes | A unique identifier generated by `crypto.randomUUID()`. Used for matching in the Options dashboard (toggle, edit, delete). |
| `path` | `string` | Yes | A CSS selector path generated by the `generateSelectorPath()` function. The algorithm prefers IDs, then stable class names (up to 2), then `:nth-of-type()` fallback. Paths can be manually edited in the Options dashboard. |
| `enabled` | `boolean` | Yes | When `true`, this selector is applied on page load and by the mutation observer. When `false`, the selector is saved but not applied. Toggling via selection mode or context menu sets this to `false` when removing RTL. |
| `createdAt` | `string` | Yes | ISO 8601 date string. Generated by `new Date().toISOString()` at the time the selector is first created. |

## URL Normalization

Page URLs are stored as the result of `window.location.origin + window.location.pathname`. This means:

- `https://example.com/page` — stored as `https://example.com/page`
- `https://example.com/page?q=hello#section` — stored as `https://example.com/page` (query and hash are dropped)
- `https://example.com/` — stored as `https://example.com/`

Two pages with different query parameters or hash fragments share the same storage entry because the URL key is identical.

## Storage Migration

In earlier versions, `rtl_data` was stored as an **object** (keyed by URL) rather than an array. The current code includes a one-time migration that converts the old format to the new array format:

```javascript
// Old format (deprecated):
{
  "rtl_data": {
    "https://example.com": { url: "...", pageEnabled: true, ... },
    "https://other.com":   { url: "...", pageEnabled: false, ... }
  }
}

// New format (current):
{
  "rtl_data": [
    { url: "https://example.com", pageEnabled: true, ... },
    { url: "https://other.com",   pageEnabled: false, ... }
  ]
}
```

The migration runs automatically in both `content.js` and `options.js` inside the `loadAllData()` function. When an object is detected, `Object.values()` is called and the result is saved back as an array. A console message `[RTL] Storage migrated from object to array format` is logged.

## Export Format

The **Export** button in the Options dashboard downloads a JSON file with the following structure:

```json
{
  "rtl_data": [
    {
      "url": "https://example.com/docs",
      "pageEnabled": true,
      "createdAt": "2025-03-15T14:30:00.000Z",
      "selectors": [
        {
          "id": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
          "path": "div.content > p.text",
          "enabled": true,
          "createdAt": "2025-03-15T14:31:00.000Z"
        }
      ]
    }
  ]
}
```

The file is named `rtl-backup-YYYY-MM-DD.json` where the date is the current UTC date.

## Import Validation

When importing a JSON file, the following validation rules are enforced:

### Top-Level Checks

- The file must parse as valid JSON.
- The root must be a JSON object (not an array or primitive).
- A top-level key `"rtl_data"` must exist.
- `"rtl_data"` must be an array.

### Page Validation

| Rule | Error Message |
|---|---|
| Page must be an object | `Page #N: must be an object` |
| `url` must be a non-empty string | `Page #N: "url" must be a non-empty string` |
| `url` must be a valid URL | `Page #N: invalid URL ("...")` |
| `pageEnabled` must be boolean | `Page #N: "pageEnabled" must be true or false` |
| `createdAt` (if present) must be a valid date | `Page #N: "createdAt" is not a valid date` |
| `selectors` must be an array | `Page #N: "selectors" must be an array` |

### Selector Validation

| Rule | Error Message |
|---|---|
| Selector must be an object | `Selector #N: must be an object` |
| `id` must be a non-empty string | `Selector #N: "id" must be a non-empty string` |
| `path` must be a non-empty string | `Selector #N: "path" must be a non-empty string` |
| `enabled` must be boolean | `Selector #N: "enabled" must be true or false` |
| `createdAt` (if present) must be a valid date | `Selector #N: "createdAt" is not a valid date` |

Validation stops reporting after 10 errors. An empty array is allowed but produces a warning: `"The file contains no pages (empty array)"`.

## Data Lifecycle

### Creation

A new `Page` entry is created when:
- The user applies RTL to an element on a page that has no existing entry. A page object with `pageEnabled: true` and one selector is saved.

A new `Selector` entry is created when:
- The user applies RTL to an element whose generated selector path does not already exist in the page's selectors array.

### Re-enable

If the user applies RTL to an element whose selector path already exists in the page's data (but is disabled), the existing selector's `enabled` flag is set back to `true` instead of creating a duplicate.

### Deletion

- **Single selector**: Deleted from the Options dashboard. If the last selector on a page is deleted, the entire page entry is removed from storage.
- **Entire page**: Deleted from the Options dashboard (both the page-level delete button and the card delete button).
- **Clear All**: Removes all data by saving an empty array `[]` to `rtl_data`.

### Soft Disable

- `Ctrl` + `Shift` + `R` sets `enabled: false` on **all** selectors for the current page, but does not delete them.
- The page-level toggle (`pageEnabled: false`) prevents selectors from being applied, but does not change their individual `enabled` flags.
